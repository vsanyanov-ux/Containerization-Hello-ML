# Контейнеризация Hello, ML! 🐳

Простое демонстрационное Machine Learning приложение, упакованное в Docker-контейнер. Этот проект служит примером базовой контейнеризации Python-скрипта, использующего популярные библиотеки для работы с данными (`numpy`, `scikit-learn`).

## 📋 Описание челленджа

Приложение делает следующие вещи:
1. Импортирует `numpy` и `scikit-learn`.
2. Создает небольшую случайную матрицу `10x3` с помощью `numpy`.
3. Создает экземпляр базового класса `sklearn.linear_model.LinearRegression` (без фактического обучения, только для демонстрации импорта).
4. Выводит на экран сообщение: `Hello from a containerized ML app!`

В проекте используется оптимизированный процесс сборки Docker-образа с помощью легковесного базового образа (`python:3.9-slim-buster`) и применения `.dockerignore` для уменьшения размера итогового образа.

## 🗂 Структура файлов

- `app.py` — Главный Python-скрипт с логикой приложения.
- `requirements.txt` — Список зависимостей (`numpy`, `scikit-learn`).
- `Dockerfile` — Инструкции для сборки Docker-образа.
- `.dockerignore` — Список файлов и папок, которые игнорируются при копировании в контекст сборки Docker.

## 🚀 Как запустить локально с помощью Docker

### 1. Предварительные требования
Убедитесь, что у вас установлен [Docker Desktop](https://www.docker.com/products/docker-desktop/).

### 2. Клонирование репозитория
```bash
git clone https://github.com/vsanyanov-ux/Containerization-Hello-ML.git
cd Containerization-Hello-ML
```

### 3. Сборка Docker-образа
Соберите образ с тегом `hello-ml-app`:
```bash
docker build -t hello-ml-app .
```

### 4. Запуск контейнера
Запустите собранный контейнер (с флагом `--rm` для автоматического удаления контейнера после завершения работы):
```bash
docker run --rm hello-ml-app
```

В результате вы должны увидеть вывод созданной матрицы, информацию об объекте регрессии и финальное приветственное сообщение.
